# Project Status Skill — Sorathel

## When to Use
Call `/status` at the start of any session, or whenever you need to see where the project stands.

## What This Skill Does
Reads all canonical files and produces a structured overview of the project, organized by the core elements of novel craft. Tells you exactly what's done, what's pending, and what to work on next.

## Process

### 1. READ THESE FILES
- `bible/story-bible.md` (what's on page)
- `bible/season-1-outline.md` (locked structure)
- `bible/anomaly-bible.md` (author-level truth)
- `drafts/revision-analysis.md` (current revision plan)
- `changelog.md` (recent activity)
- All files in `characters/` (character sheet status)
- All files in `episodes/` (what's written)

### 2. REPORT STRUCTURE

Produce the status report using these sections. Each section should be concise — bullets, not paragraphs.

---

#### A. PROSE STATUS
*What exists as written text.*
- List every chapter: file path, POV character, word count, draft status (first draft / revised / final)
- Total word count across all chapters
- Current episode progress (e.g., "Episode 1: 3 of ~5 chapters written")

#### B. PLOT — Promises, Progress, Payoffs
*Sanderson's framework: what has the story promised the reader? How far along are those promises? Which need payoff soon?*

**Promises Made** (things the text has set up that the reader expects to see resolved):
- List every planted seed from the Story Bible's "Planted Seeds" table
- Mark each as: PLANTED / PROGRESSING / PAID OFF / OVERDUE

**Major Dramatic Question:**
- State the core question and where the reader currently sits relative to it

**Big P Plot** (overarching story arc):
- Current position in the season structure (which act, which episode)

**Little p Plot** (immediate chapter-level tensions):
- What was the last chapter's ending tension?
- What does the next chapter need to address?

#### C. CHARACTER STATUS
*For each character who has appeared or is planned:*

| Character | On Page? | Sheet Complete? | Arc Position | Voice Tested? | Needs Work? |
|-----------|----------|-----------------|--------------|---------------|-------------|

For each POV character, also note:
- **Try-Fail Cycles**: What has this character tried and failed at? What's their next attempt?
- **Internal Progress**: Where are they in their internal arc? What belief is being questioned?
- **Contradiction Beat**: Has each chapter shown at least one moment contradicting their dominant trait? (per Writing Rule)

#### D. SETTING & WORLDBUILDING
*What's been shown on page vs. what's defined but not yet shown.*
- **Locations shown**: List with chapter references
- **Locations defined but not shown**: List from outlines/reference docs
- **Creature bonds shown**: What the reader knows about the bond system
- **Cultural details established**: Food, currency, customs, architecture
- **Terminology introduced in prose**: Which terms the reader has encountered naturally

#### E. MAGIC SYSTEM
*Sanderson's Laws applied to the Amarvel system.*
- **What the reader understands** (shown through action, not exposition):
  - Pattern reading, branch work, deep reading cost, bond behavior, node mechanics
- **What's defined but not shown**: List mechanics from the Anomaly Bible that haven't appeared in prose yet
- **Sensory vocabulary established**: What does magic look/feel/sound like on the page? (per GRRM Fix Plan concern)
- **Cost/limitation shown**: What prices has the reader seen paid?

#### F. PACING & STRUCTURE
- **Current act position**: Act 1 / Act 2 / Act 3
- **Words written vs. target**: X of 180,000-220,000
- **Average chapter length**: X words (target: 3,500-5,500)
- **POV distribution so far**: Vaelen X% / Drenna X% / Caeleth X% / Zethael X% (target: Vaelen 60%, Drenna 30%, Zethael 10%)
- **Tone escalation**: Is the wrongness building chapter to chapter? Where on the spectrum? (beautiful → threatened → broken)

#### G. REVISION STATUS
- **Chapters needing revision**: List with summary of what needs changing (from revision-analysis.md)
- **Writing rules compliance**: Any known violations in current drafts?
- **Continuity issues**: Any known contradictions between chapters?

#### H. REFERENCE COMPLETENESS
- **Story Bible**: Up to date through which chapter?
- **Character sheets**: Which are complete vs. placeholder?
- **Episode outlines**: Which episodes have chapter-level outlines?
- **Anomaly Bible**: Current version, last updated

#### I. RECOMMENDED NEXT ACTION
Based on all the above, suggest ONE clear next step. Prioritize:
1. Revision of existing chapters (if revision analysis exists and hasn't been done)
2. Writing the next chapter in sequence
3. Outlining the next episode (if current episode is complete)
4. Filling gaps in character sheets or world-building (only if blocking prose work)

State the recommendation clearly: "**Next: [action]** — because [reason]."

---

## Format Notes
- Keep the entire report scannable — no prose paragraphs, use tables and bullet lists
- Bold the most important items (overdue payoffs, blocking issues, the next action)
- If something is in good shape, say so briefly and move on — don't pad
- Total report should fit in one screen (~100-150 lines)
